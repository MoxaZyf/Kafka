# Система обработки сообщений с блокировкой пользователей и цензурой

## Описание проекта

Реализован потоковый сервис обмена сообщениями на базе **Apache Kafka** и **Faust** (Python).  
Система обеспечивает:

- **Блокировку нежелательных пользователей** – получатель может заблокировать отправителя, после чего сообщения от него игнорируются.
- **Динамическую цензуру** – список запрещённых слов можно обновлять без перезапуска; все сообщения проходят фильтрацию перед доставкой.

## Архитектура и компоненты

### Топики Kafka

| Топик               | Тип сообщения       | Назначение                                      |
|---------------------|---------------------|-------------------------------------------------|
| `messages`          | `Message`           | Входящие сообщения (отправитель, получатель, текст) |
| `filtered_messages` | `FilteredMessage`   | Сообщения после проверки блокировок и цензуры   |
| `blocked_users`     | `BlockAction`       | События блокировки/разблокировки (user_id, blocked_user, action) |
| `banned_words`      | `BannedWordUpdate`  | Добавление/удаление запрещённых слов            |

### Модели данных (Faust Records)

- **`Message`** – исходное сообщение.
- **`FilteredMessage`** – обработанное сообщение (содержит цензурированный текст и оригинал).
- **`BlockAction`** – команда блокировки (`action="block"` или `"unblock"`).
- **`BannedWordUpdate`** – команда обновления словаря (`action="add"` или `"remove"`).

### Таблицы состояний (персистентное хранилище RocksDB)

- **`blocked_table`** – для каждого пользователя (получателя) хранит список заблокированных отправителей.
- **`banned_words_table`** – список запрещённых слов (ключ `'words'`).

### Логика работы

1. **Обработка блокировок** – агент `process_block_actions` читает топик `blocked_users` и обновляет `blocked_table`.
2. **Обработка запрещённых слов** – агент `process_banned_words` читает `banned_words` и обновляет `banned_words_table`.
3. **Основной поток** – агент `process_messages`:
   - Получает сообщение из `messages`.
   - Проверяет, не заблокирован ли отправитель получателем (по `blocked_table`). Если да – сообщение отбрасывается.
   - Применяет цензуру: заменяет каждое запрещённое слово на `***`.
   - Отправляет результат в `filtered_messages`.
